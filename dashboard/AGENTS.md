# AI 작업 규칙

## 기본 기준

- 이 폴더는 루트 스타터와 분리된 별도 대시보드 프로젝트로 본다.
- 실제 확인 대상은 `html/` 안의 HTML 파일이다.
- 작업 전 `docs/guide.md`와 `docs/progress.md`를 확인한다.
- 기존 폴더 구조와 연결된 CSS/JS 경로를 유지한다.
- 전체 파일을 재생성하지 않고 필요한 영역만 수정한다.

## 진행 기록

- 새 섹션, 구조 변경, 기능 추가를 시작하기 전 `docs/progress.md`를 확인한다.
- 의미 있는 작업이 끝나면 `docs/progress.md`를 갱신한다.
- 작은 CSS 수정, 단순 문구 수정, 여백 조정은 기록하지 않는다.
- 오래된 이력은 누적하지 않고 `마지막 작업 / 현재 상태 / 다음 작업 / 주의사항` 형식을 유지한다.

## 작업 방식

- 화면 전체를 한 번에 바꾸지 않고 섹션 또는 컴포넌트 단위로 작업한다.
- 현재 작업 중인 영역만 수정한다.
- 기존 class명, 마크업 패턴, 컴포넌트를 우선 재사용한다.
- 실제 데이터 연동 전에는 정적 상태와 mock 데이터를 기준으로 화면 구조를 맞춘다.
- 불필요한 라이브러리를 설치하지 않는다.

## HTML/CSS/JS 기준

- 목적이 분명한 `header`, `main`, `section`, `aside`, `nav`, `footer`를 우선 사용한다.
- 버튼 동작은 `button`, 페이지 이동은 `a`를 사용한다.
- 검색 input은 `label` 또는 `sr-only` label과 연결한다.
- 아이콘만 있는 버튼은 `aria-label`을 제공한다.
- `resources/css/common.css`는 reset, font, 변수, 기본 태그 스타일 용도로만 본다.
- 화면 스타일은 `resources/css/style.css`에 작성하고 별도 `layout.css`는 사용하지 않는다.
- 현재 정적 화면에 필요 없는 JS는 추가하지 않는다.
- JS가 필요하면 기능 범위를 명확히 하고 HTML 연결 여부를 확인한다.

## 상태와 검수

- 필요한 화면에서는 기본, 데이터 없음, 검색 결과 없음, 로딩, 오류, 선택, 비활성화, 권한 없음 상태를 고려한다.
- 모바일에서 텍스트, 버튼, 검색창, 탭, 페이지네이션이 영역 밖으로 넘치지 않게 한다.
- 본문 바로가기 링크와 키보드 포커스 표시를 유지한다.
- 사용하지 않는 CSS/JS 링크, 깨진 이미지 요청, 콘솔 오류를 확인한다.

## 응답 방식

- 사용자의 의견에 자동으로 동의하지 않고 타당성과 효율성을 먼저 검토한다.
- 더 나은 대안이 있으면 근거와 함께 제안한다.
- 문제가 없으면 그대로 진행해도 된다고 설명한다.
- 변경된 영역만 최소 단위로 설명한다.
