# 프로젝트 가이드

## 목적

- 소방안전 빅데이터 플랫폼의 분석도구 대시보드 화면을 정적 HTML/CSS로 관리한다.
- 기준 산출물은 `html/index.html`이다.
- 현재 단계는 실제 데이터 연동 전의 정적 화면과 mock 데이터 구성이다.


## 현재 구조

```text
dashboard/
├─ AGENTS.md
├─ docs/
│  ├─ guide.md
│  └─ progress.md
├─ html/
│  └─ index.html
└─ resources/
   ├─ css/
   │  ├─ common.css
   │  └─ style.css
   ├─ js/
   └─ fonts/
```

## 화면 구성

- `header`: 로고, 서비스명, 주요 메뉴
- `main`: 데이터셋 선택 패널과 시각화 본문 영역
- `aside.dataset_panel`: 검색, 총 건수, 데이터셋 목록, 페이지네이션
- `section.content`: 선택 안내, 빈 상태 또는 추후 차트/테이블 영역
- `footer`: 기관 표기와 copyright

## 구현 기준

- 기존 class명과 마크업 패턴을 유지한다.
- `common.css`는 불필요하게 수정하지 않는다.
- 대시보드 전용 스타일은 `style.css`에 작성한다.
- 현재 화면은 별도 JS 없이 동작하는 정적 화면을 기준으로 한다.
- 실제 데이터 연동 시 검색, 목록 선택, 페이지네이션, 로딩/빈값/오류 상태를 JS 범위로 분리한다.
- 이미지 파일은 현재 사용하지 않으며, 로고와 빈 상태 그래픽은 CSS 기반 임시 표현이다.

## 반응형 기준

- 1023px 이하: 데이터셋 패널을 상단 전체 폭으로 배치한다.
- 768px 이하: 헤더와 푸터 줄바꿈을 허용하고 주요 텍스트가 넘치지 않게 한다.
- 긴 데이터셋명은 말줄임 또는 줄바꿈 기준을 명확히 적용한다.
- 차트나 테이블이 추가되면 모바일 표시 방식을 먼저 정한다.

## 접근성 기준

- 본문 바로가기 링크를 유지한다.
- 주요 기능은 키보드 Tab 이동으로 접근 가능해야 한다.
- 포커스 표시를 제거하지 않는다.
- 현재 선택된 메뉴, 목록, 페이지에는 class와 함께 `aria-current` 사용을 검토한다.
- table 추가 시 `caption`을 제공한다.
- 색상만으로 선택, 오류, 비활성화 상태를 구분하지 않는다.

## 검수 기준

- `html/index.html`에서 사용하지 않는 CSS/JS 링크가 없는지 확인한다.
- 깨진 이미지 요청과 콘솔 오류가 없는지 확인한다.
- 모바일에서 목록, 검색 입력, 페이지네이션, 안내 문구가 영역 밖으로 넘치지 않는지 확인한다.
- 데이터 없음, 검색 결과 없음, 로딩, 오류 등 필요한 상태 화면이 누락되지 않았는지 확인한다.
