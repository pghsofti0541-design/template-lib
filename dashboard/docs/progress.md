# 현재 진행 상황

## 마지막 작업

- `AGENTS.md`, `docs/guide.md`, `docs/progress.md`의 깨진 문자를 정리했다.
- 중복 규칙을 줄이고 문서 역할을 분리했다.
- 현재 대시보드 기준과 진행 기록 형식을 간소화했다.

## 현재 상태

- 기준 산출물은 `html/index.html`이다.
- 화면은 보유 데이터셋 목록과 시각화 초기 안내가 있는 정적 HTML/CSS 상태다.
- HTML에는 `common.css`, `style.css`만 연결되어 있다.
- 별도 JS 동작과 실제 데이터 연동은 아직 적용하지 않았다.
- 이미지 파일은 사용하지 않고 CSS 기반 그래픽을 사용한다.

## 다음 작업

1. 브라우저에서 `html/index.html` 렌더링과 콘솔 오류를 확인한다.
2. 화면 문구와 HTML 인코딩 상태를 점검한다.
3. 실제 데이터 연동 전 API 응답 구조와 상태 화면 범위를 정의한다.

## 주의사항

- `resources/css/common.css`는 불필요하게 수정하지 않는다.
- 대시보드 전용 CSS는 `resources/css/style.css`에 작성한다.
- 새 기능을 추가할 때만 JS 파일 분리와 HTML 연결을 검토한다.
- 루트 스타터의 메인/서브 스타일을 이 프로젝트에 그대로 적용하지 않는다.
